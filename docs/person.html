Athena Knowledgebase

A conversational Agent

Präfix

/persons

Input:

Keiner.

Output:

Gibt die kompletten Daten aller Personen zurück.

Spezifische Anfragen

/byPersonID/{value}

Input:
"value":

Beschreibt die datenbankinterne, einzigartige ID einer Person. Diese muss eine Zahl größer gleich 1 sein, ohne führende Null.
Beispiel: 5 oder 42

Output:

Gibt die kompletten Daten der Person zurück, die die ID hat, die beim Aufrufen des APIs angegeben wurde.

/byPersonID/{value}/getPapers

Input:
"value":

Beschreibt die datenbankinterne, einzigartige ID eines Autors. Diese muss eine Zahl größer gleich 1 sein, ohne führende Null.
Beispiel: 5 oder 42

Output:

Gibt die kompletten Daten aller Paper zurück, an denen der Autor mit der ID, die beim Aufrufen des APIs angegeben wurde, mitgewirkt hat.

/byPrefix/{value}

Input:
"value":

Beschreibt den (akademischen) Titel einer Person. Dieser muss exakt sein, Leerzeichen und Sonderzeichen sind erlaubt.
Beispiel: Prof. Dr. oder Dipl. Ing.

Output:

Gibt die kompletten Daten aller Personen zurück, die den (akademischen) Titel innehaben, der beim Aufrufen des APIs angegeben wurde.

/byFullName/{value}

Input:
"value":

Beschreibt den vollen Namen einer Person. Dieser muss exakt sein, Leerzeichen und Sonderzeichen sind erlaubt.

Output:

Gibt die kompletten Daten der Personen zurück, die den Namen haben, der beim Aufrufen des APIs angegeben wurde.

/byBirthdate/{year}/{month}/{day}

Input:
"year":

Beschreibt das Jahr, in dem eine Person geboren worden ist. Das Jahr muss das Format yyyy haben und darf nur aus Ziffern bestehen.
Beispiel: 2018

"month":

Beschreibt den Monat, in dem eine Person geboren worden ist. Der Monat muss das Format mm haben und darf nur aus Ziffern bestehen.
Beispiel für Juli: 07 oder 7

"day":

Beschreibt den Tag, an dem eine Person geboren worden ist. Der Tag muss das Format dd haben und darf nur aus Ziffern bestehen.
Beispiel: 01, 1 oder 15

Output:

Gibt die kompletten Daten aller Personen zurück, die an dem Datum geboren worden sind, das beim Aufrufen des APIs angegeben wurde.
