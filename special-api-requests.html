Special API Requests

Get amount of results

Any API request that is not listed on this page can be prepended with "/count" to receive the size of the result set.

Example: /count/paper:paperID=1/person will output the amount of authors that worked on the paper with id 1.

Keep in mind, that this will not show you the actual data, only the size of the result set.


StreetMap Data

The request is mapped to

/openStreetMaps/location/minLatitude/{minLatitude}/minLongitude/{minLongitude}/maxLatitude/{maxLatitude}/maxLongitude/{maxLongitude}/amenity/{amenity}/radiusInMeter/{radius}")

Because GPS data is rarely exact, the API functions with a concept known as bounding box. We specify a range of longitudes and latitudes respectively, yielding a box from which the GPS signal originates.

  • Longitudes allow values from -180 to 180
  • Latitudes allow values from -90 to 90

Next, we specify the type of amenity we are interested in. The possibilities are large and include values such as "restaurant", "toilets" or even "tree". Play around and see what you find.

The last value to specify is the searchradius, in meters. Try to keep this as small as reasonable, in order to limit strain on the Overpass Server (where we get our data from).

This is a working example that returns a list of Locations (restaurants in this case):

/openStreetMaps/location/minLatitude/48.5657094/minLongitude/13.4490548/maxLatitude/48.5662416/maxLongitude/13.4501676/amenity/restaurant/radiusInMeter/3000

Location

Every location returned always features the following attributes:

  • An id (long)
  • longitude (double) The longitude of this location
  • latitude (double) The latitude of this location
  • type (usually the String "node")
  • amenity (String) - the type of amenity of this location
