Präfix

/events

Input:

Keiner.

Output:

Gibt die kompletten Daten aller Events zurück.

Spezifische Anfragen

/byEventID/{value}

Input:
"value":

Beschreibt die datenbankinterne, einzigartige ID eines Events. Diese muss eine Zahl größer gleich 1 sein, ohne führende Null.
Beispiel: 5 oder 42

Output:

Gibt die kompletten Daten des Events zurück, das die ID hat, die beim Aufrufen des APIs angegeben wurde.

/byEventID/{value}/getSessions

Input:
"value":

Beschreibt die datenbankinterne, einzigartige ID eines Events. Diese muss eine Zahl größer gleich 1 sein, ohne führende Null.
Beispiel: 5 oder 42

Output:

Gibt die kompletten Daten aller Sessions zurück, die während des Events mit der ID, die beim Aufrufen des APIs angegeben wurde, als Teil dessen stattfinden.

/byConference/{value}

Input:
"value":

Beschreibt den Namen einer Konferenz. Dieser muss exakt sein, Leerzeichen und Sonderzeichen sind erlaubt.

Output:

Gibt die kompletten Daten aller Events zurück, die während der Konferenz stattgefunden haben, deren Name beim Aufrufen des APIs angegeben wurde.

/byDate/{year}/{month}/{day}

Input:
"year":

Beschreibt das Jahr, in dem ein Event stattgefunden hat. Das Jahr muss das Format yyyy haben und darf nur aus Ziffern bestehen.
Beispiel: 2018

"month":

Beschreibt den Monat, in dem ein Event stattgefunden hat. Der Monat muss das Format mm haben und darf nur aus Ziffern bestehen.
Beispiel für Juli: 07 oder 7

"day":

Beschreibt den Tag, an dem ein Event stattgefunden hat. Der Tag muss das Format dd haben und darf nur aus Ziffern bestehen.
Beispiel: 01, 1 oder 15

Output:

Gibt die kompletten Daten aller Events zurück, die an dem Datum stattgefunden haben, das beim Aufrufen des APIs angegeben wurde.

/byStartTime/{hour}/{minute}

Input:
"hour":

Beschreibt die Stunde, in der ein Event gestartet ist. Die Stunde muss zwischen 0 und 23 (beide einschließlich) liegen und darf nur aus Ziffern bestehen.
Beispiel: 01, 1 oder 15

"minute":

Beschreibt die Minute, in der ein Event gestartet ist. Die Minute muss zwischen 0 und 59 (beide einschließlich) liegen und darf nur aus Ziffern bestehen. Beispiel: 01, 1 oder 15

Output:

Gibt die kompletten Daten aller Events zurück, die zu der Zeit gestartet sind, die beim Aufrufen des APIs angegeben wurde.

Beschreibt die Minute, in der ein Event geendet hat. Die Minute muss zwischen 0 und 59 (beide einschließlich) liegen und darf nur aus Ziffern bestehen. Beispiel: 01, 1 oder 15

Output:

Gibt die kompletten Daten aller Events zurück, die zu der Zeit geendet haben, die beim Aufrufen des APIs angegeben wurde.

/byCategory/{value}

Input:
"value":

Beschreibt die Kategorie eines Events. Diese muss eine der in der Liste angegebenen Ziffern sein.
Liste an Kategorien:
0: BREAK
1: CEREMONY
2: MEETING
3: PRESENTATION
4: RECRUITMENT
5: SESSION
6: SOCIAL
7: TALK
8: TUTORIAL
9: WELCOME
10: WORKSHOP

Output:

Gibt die kompletten Daten aller Events zurück, die in der Kategorie sind, die beim Aufrufen des APIs angegeben wurde.
