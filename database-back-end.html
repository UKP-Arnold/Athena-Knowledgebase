Database Backend

The database back end is detached from our REST API through interfaces. Their reference implementation uses JPA with Hibernate.

JPA Configuration

All JPA Configuration is located in the ./src/main/resources/META-INF/persistence.xml-File. It contains configurtaion for persistence units to be bootstrapped and used by JPA. You can add and edit persistence units here. With those units you can switch the database back ends used by JPA.

EntityManagerFactory

Note that JPA uses EntityManager-Objects to communicate with the database. Those are provided from an EntityManagerFactory which needs to be initialized with a persistence unit (see above). Note that choice of persistence units happens while initializing the EntityManagerFactory.

In our reference implementation you can find it in the de.tudarmstadt.informatik.ukp.athenakp.database.jpa.PersistenceManager class.

Other Database Back Ends

Switching from JPA to other database back ends involves a lot more work than just switching the bootstrapped persistence unit of JPA. It requires initialization of the new back end and implementation of all interfaces located in the de.tudarmstadt.informatik.ukp.athenakp.database.access package.
